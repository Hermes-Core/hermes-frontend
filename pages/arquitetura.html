Modelo Arquitetural

Infraestrutura e Tecnologia

O sistema Hermes foi concebido sob a ótica da escalabilidade, segurança e alta disponibilidade, utilizando um modelo totalmente desacoplado entre o cliente e o servidor.

Visão Geral

Estrutura em Camadas

A arquitetura divide-se em componentes independentes para facilitar a manutenção e evolução do ecossistema.

01

Frontend

Desenvolvido como uma Single Page Application (SPA) utilizando React e Vite. É responsável pela interface, gerenciamento do estado local e renderização dinâmica para smartphones e computadores. Hospedado na Vercel.

02

Backend

Construído em Java com o framework Spring Boot. Segue o padrão de controladores, serviços e repositórios, centralizando as regras de negócio, segurança (RBAC) e validação de dados. Hospedado no Railway.

03

Banco de Dados

Persistência relacional utilizando o PostgreSQL Serverless através da infraestrutura Neon. Garante a integridade referencial e segurança total dos dados acadêmicos e registros dos estudantes.

Integração

APIs e Comunicação

Como as partes do sistema conversam entre si de forma assíncrona e segura.

REST

Troca de Dados em JSON

O Frontend e o Backend se comunicam exclusivamente via requisições HTTP RESTful, trafegando dados estruturados no formato JSON.

ORM

Spring Data JPA

A ponte entre o servidor de aplicação e a base de dados PostgreSQL é feita de forma segura por mapeamento objeto-relacional.

JWT

Segurança Baseada em Tokens

O tráfego de dados sensíveis exige autenticação com tokens (evoluindo para JWT), protegendo a privacidade dos alunos e responsáveis.

Visualização

Diagrama de Camadas

Representação visual do desacoplamento físico e fluxo de dados do ecossistema Hermes.
